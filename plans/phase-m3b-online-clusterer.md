# Phase M3b: Online assign-or-spawn clustering engine

**Milestone:** M3 — Global clustering engine + classification (the core)
**Status:** Not started
**Estimated effort:** M

## Goal
A day's deduped candidate articles collapse into rolling story clusters — each candidate either **joins** the nearest existing cluster (cosine ≥ τ_assign, centroid updated by running mean) or **spawns** a new one — and a cluster seen on a prior day is re-matched today with a **stable story id** (cross-day continuity via `story_url_aliases`). Composes the M3a primitives (`embeddings`, `near_dup`, `cluster_store`); persists to the live `story_clusters` / `story_cluster_members` tables (migration 0018, already applied). Classification of the centroid into a category/subcategory and Stage-B ingest are **M3c**, not here.

## Context for the executor (read before coding)
- **M3a is shipped and tested** in `agents/pipeline/clustering/`: `embeddings.embed_texts(texts, *, llm_client) -> list[list[float]]` (768-d, L2-normalized) and `embeddings.cosine_similarity(a, b)` (dot product); `near_dup.drop_exact_reprints(items: list[NearDupItem], *, threshold=0.7) -> list[int]` (keeps one representative index per reprint group) and `near_dup.NearDupItem(item_index, item_text)`; `cluster_store.load_active_clusters(client, *, category=None, since_utc)`, `upsert_cluster(client, cluster)`, `add_cluster_members(client, cluster_id, members)`; `models.StoryCluster` / `models.ClusterMember` (mirror the 0018 columns; `cluster_centroid: list[float] | None`).
- **The persisted schema (0018) has NO entity/theme column.** Blocking is therefore **time-window + provisional category**, and the cosine match against `cluster_centroid` does the actual same-event decision (spec §2C step 4). Do not invent an entity/theme table.
- **`cluster_category` is `not null`** in the DDL, so a spawned cluster needs a provisional category at spawn time. M3b uses a caller-supplied provisional category (e.g. the candidate's matched-interest root); **M3c's centroid classifier overwrites it** with the authoritative `(category, subcategory)`. Reel-format stays the DDL default `'event'` (M5 sets real formats).
- **Cross-day continuity** reuses `daily_batch.build_story_id_resolver(client) -> (normalized_urls) -> {normalized_url: existing_story_id}` (chunked `.in_()` over `story_url_aliases`). M3b consumes that **callable contract** (injected), it does not re-implement alias lookup.
- **τ_assign default = 0.75** (owner execution delta, spec §3 banner; tunable, real-corpus tuning deferred to M6). Window default = 48h (spec §2C step 3).
- **Every test mocks the Gemini + supabase boundaries** (CLAUDE.md §6) — no real API/DB calls, no cost. Migration 0018 is already live; this phase applies no migration.

## Sub-phases

### Sub-phase 1: Engine I/O models + time-window blocking
- **Files touched:** `agents/pipeline/clustering/engine_models.py` (new), `agents/pipeline/clustering/blocking.py` (new), `tests/agents/pipeline/clustering/test_blocking.py` (new).
- **What ships:** `engine_models.py` — pydantic `ClusterInput` (`input_index: int`, `input_text: str` (headline+lead, for shingling+embedding), `input_url: str`, `input_outlet: str | None`, `input_published_utc: datetime`, `input_provisional_category: str`) and the run-result holder `ClusterRun` (`clusters: list[StoryCluster]`, `members: list[ClusterMember]`, `input_cluster_map: dict[int, str]` mapping a kept input_index → its cluster_id). `blocking.py` — pure `select_block(active_clusters: list[StoryCluster], *, candidate_published_utc: datetime, window_hours: int = 48, category: str | None = None) -> list[StoryCluster]` returning clusters whose `cluster_last_seen_utc` is within `window_hours` of the candidate (and matching `category` when given), so the assign step is O(block) not O(n²).
- **What ships (cont.):** a `DEFAULT_WINDOW_HOURS = 48` constant.
- **Definition of done:** `test_blocking.py` (pure, no mocks needed) asserts: (a) a cluster `last_seen` 47h before the candidate is IN the block, one at 49h is OUT (window math correct, both sides of the boundary); (b) passing `category="tech"` excludes a `"sport"` cluster in-window; (c) exactly-at-window-edge (48h) behavior is asserted explicitly (documented inclusive/exclusive); (d) `ClusterInput`/`ClusterRun` validate a minimal example. `pytest tests/agents/pipeline/clustering/test_blocking.py -q` green; `ruff check` clean.
- **Dependencies:** none

### Sub-phase 2: Assign-or-spawn decision + running-mean centroid update
- **Files touched:** `agents/pipeline/clustering/assign.py` (new), `tests/agents/pipeline/clustering/test_assign.py` (new).
- **What ships:** pure functions over plain vectors + `StoryCluster` (no dependency on SP1): `best_match(candidate_embedding: list[float], block_clusters: list[StoryCluster]) -> tuple[StoryCluster | None, float]` (max `cosine_similarity` against each cluster's centroid, skipping null-centroid clusters); `should_assign(score: float, *, tau_assign: float = DEFAULT_TAU_ASSIGN) -> bool`; `update_centroid_running_mean(old_centroid: list[float], old_count: int, new_embedding: list[float]) -> list[float]` (incremental mean `((old*n)+new)/(n+1)` then **re-L2-normalize** so cosine stays a dot product). `DEFAULT_TAU_ASSIGN = 0.75`.
- **Definition of done:** `test_assign.py` asserts the *why*, not just the math: (a) given a block where one cluster centroid is cos ≈0.9 and another ≈0.2 to the candidate, `best_match` returns the 0.9 cluster with its score; (b) `should_assign` is True at 0.75 and just above, False just below (τ boundary encodes the join-vs-spawn business rule); (c) running-mean of three unit vectors added one at a time equals the batch mean direction (the centroid tracks its members, not just the last one) and the result is L2-normalized (‖v‖≈1); (d) `best_match([], ...)` and all-null-centroid block return `(None, ...)` so the orchestrator spawns. `pytest tests/agents/pipeline/clustering/test_assign.py -q` green; `ruff check` clean.
- **Dependencies:** none (parallel with Sub-phase 1)

### Sub-phase 3: Online clustering orchestrator
- **Files touched:** `agents/pipeline/clustering/online_clusterer.py` (new), `tests/agents/pipeline/clustering/test_online_clusterer.py` (new).
- **What ships:** `async def cluster_candidates(candidates: list[ClusterInput], *, llm_client, existing_clusters: list[StoryCluster], mint_cluster_id: Callable[[], str], tau_assign: float = DEFAULT_TAU_ASSIGN, window_hours: int = DEFAULT_WINDOW_HOURS) -> ClusterRun`. Pipeline per spec §2C steps 1→4: (1) **near-dup prefilter** — build `NearDupItem`s from `input_text`, keep `drop_exact_reprints` representatives (dropped reprints still become `ClusterMember`s of the rep's cluster); (2) **embed** the representatives via `embed_texts`; (3) for each rep, `select_block(existing+spawned this run, ...)` then `best_match`/`should_assign`; (4) **join** → append member, bump `cluster_member_count`/`cluster_outlet_count` (distinct outlet), advance `cluster_last_seen_utc`, `update_centroid_running_mean`; **spawn** → mint id, new `StoryCluster` (provisional category from the input, centroid = the embedding, counts=1). Returns the `ClusterRun`.
- **Definition of done:** `test_online_clusterer.py` mocks `embed_texts` (patched to return deterministic unit vectors keyed by input text — **no real Gemini**) and injects a counter `mint_cluster_id`; asserts: (a) two byte-near-duplicate inputs collapse to ONE cluster with two members (near-dup prefilter wired); (b) two inputs whose mocked embeddings are cos≥τ land in ONE cluster and the centroid moved to their mean; (c) an input with an orthogonal embedding **spawns** a second cluster; (d) an input matching a passed-in `existing_clusters` centroid (≥τ, in-window) **joins it across the day boundary** (no new id minted) and advances its `last_seen`/`member_count`; (e) `cluster_outlet_count` counts distinct outlets. `pytest tests/agents/pipeline/clustering/test_online_clusterer.py -q` green; `ruff check` clean.
- **Dependencies:** Sub-phase 1 (`ClusterInput`/`ClusterRun`, `select_block`), Sub-phase 2 (`best_match`/`should_assign`/`update_centroid_running_mean`)

### Sub-phase 4: Cross-day id bridge + persistence wiring
- **Files touched:** `agents/pipeline/clustering/continuity.py` (new), `agents/pipeline/clustering/online_clusterer.py` (extend), `tests/agents/pipeline/clustering/test_continuity.py` (new).
- **What ships:** `continuity.py` — `resolve_cluster_story_ids(run: ClusterRun, *, resolve_existing_story_ids: Callable[[list[str]], dict[str, str]], mint_story_id: Callable[[], str]) -> dict[str, str]` mapping each `cluster_id → story_id`: normalize each cluster's member URLs, call the injected resolver (the `build_story_id_resolver` callable contract) once, and **if any member already aliases to an existing `story_id`, reuse it** (a multi-day event keeps its id → produce-once/don't-repeat stay correct); else `mint_story_id()`. Plus `persist_run(client, run: ClusterRun) -> None` calling `cluster_store.upsert_cluster` + `add_cluster_members` per cluster (injected client). A thin `online_clusterer` wrapper exposes the run→resolve→persist sequence so the M3c batch can call one entry point.
- **Definition of done:** `test_continuity.py` (resolver + supabase client MOCKED) asserts: (a) a cluster whose member URL the mocked resolver maps to story `S` gets `story_id == S` (cross-day continuity — the load-bearing reuse), and `mint_story_id` is NOT called for it; (b) a cluster with only unseen URLs mints a fresh id; (c) `persist_run` calls `upsert_cluster` once per cluster and `add_cluster_members` with that cluster's members (assert call args, mocked client — **no live DB**); (d) URL normalization matches the alias key form used by `build_story_id_resolver` (same normalization, else continuity silently misses). `pytest tests/agents/pipeline/clustering/test_continuity.py -q` green; `ruff check` clean.
- **Dependencies:** Sub-phase 3 (`ClusterRun`, the orchestrator)
