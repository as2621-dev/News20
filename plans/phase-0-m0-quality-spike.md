# Phase 0: M0 Quality-Spike — render 5 real digests

**Milestone:** M0 — Quality spike (de-risk gate)
**Status:** Not started
**Estimated effort:** M

## Goal
Five real, finished `.mp4` anchor-duo digests exist on disk — built from the already-written scripts + image prompts in `documents/m0-digests.md` using the real reusable pipeline pieces (TLDW TTS + forced-alignment captions + a new Remotion 9:16 compositor) — ready to put in front of strangers for the "would you watch 10 in a row?" go/no-go gate before any M1 build.

## Context the sub-agents need
- **Scripts + shot lists are already authored** in `documents/m0-digests.md` (5 digests, each 8 cuts ≈ 7s, speaker-tagged ALEX/JORDAN lines, per-cut caption keyword, per-cut image prompt). Do not re-write them.
- **Reuse over new (Rule 2 + `reference/reuse-map.md`):** TLDW donor lives at `~/TLDW-Phase2/tldw/voice-agent-dashboard/`. Read the donor file before porting (Rule 8); copy then adapt, don't hand-rewrite from memory. Do **not** edit anything under TLDW `_legacy/`.
- **Locked format (`documents/m0-digests.md` + `reference/stack-notes.md`):** ~55s, single-source, two hosts. ALEX → Gemini voice `Leda`, JORDAN → Gemini voice `Sadaltager`, model `gemini-2.5-flash-preview-tts`, multi-speaker single call, chunk on `<Person[12]>` boundaries (~4000-byte budget). Captions: bold white, black outline, lower-middle third, word-by-word reveal, one `#FACC15` highlight word per sentence. 9:16, 8 cuts, Ken Burns motion, no AI video, no avatars.
- **Render manifest is the seam** between the Python pipeline and Remotion (`reference/stack-notes.md`): `{ audio_path, ordered images + Ken Burns params, caption timing (from forced alignment), headline-card data }`. TLDW `agents/pipeline/stages/tts_handoff.py` is the handoff template.
- **Conventions (`reference/conventions.md`):** verbose prefixed names; structured-JSON logging via `agents/shared/logger.py` (`fix_suggestion` on every error log); Pydantic v2 at boundaries; Ruff (line length 120); keys only in `.env`, keep `.env.example` current; agent files <500 LoC.

## Sub-phases

### Sub-phase 1: Port the TTS spine and render the 5 digests' audio
- **Files touched:** `agents/shared/{logger,settings,exceptions}.py` (PORT from TLDW), `agents/voice/{gemini_tts,audio,models}.py` (PORT), `agents/m0/digests_input.py` (NEW — the 5 scripts as ordered speaker-tagged turns, transcribed from `documents/m0-digests.md`), `agents/m0/render_audio.py` (NEW driver), `requirements.txt` + `.env.example` (PORT, drop yt-dlp/youtube deps), `tests/agents/voice/test_gemini_tts.py` (NEW).
- **What ships:** 5 assembled audio files at `agents/m0/output/audio/digest-{1..5}.<ext>`, each ~50–60s, ALEX lines synthesized as `Leda` and JORDAN lines as `Sadaltager`, via chunked multi-speaker calls. `agents/m0/digests_input.py` exposes the 5 scripts as typed turn lists reusable by sub-phases 2 and 4.
- **Definition of done:** Running `python -m agents.m0.render_audio` writes 5 audio files; `ffprobe` reports each duration in 45–70s. A unit test mocks the Gemini TTS client and asserts (a) ALEX→`Leda` / JORDAN→`Sadaltager` mapping at the call boundary, (b) text is chunked on `<Person[12]>` boundaries under the byte budget, (c) the assembled turn order matches the script. Ruff passes; no key value is logged.
- **Dependencies:** none

### Sub-phase 2: Port forced alignment → word-by-word caption tracks
- **Files touched:** `agents/pipeline/stages/forced_alignment.py` (PORT), its minimal immediate deps only if required (`agents/pipeline/{models,json_utils}.py` — PORT, surgical), `agents/m0/align_captions.py` (NEW driver), `tests/agents/pipeline/test_forced_alignment.py` (NEW). Output: `agents/m0/output/captions/digest-{1..5}.captions.json`.
- **What ships:** Per digest, a caption-track JSON of `{word, start_s, end_s}` entries plus the per-sentence highlight keyword (taken from the "caption keyword" column in `documents/m0-digests.md`), conforming to the caption-track shape the Remotion composition consumes (see sub-phase 3).
- **Definition of done:** Running `python -m agents.m0.align_captions` produces 5 caption JSONs. A unit test asserts (a) every word timing is monotonic and non-overlapping, (b) all timings fall within the digest's audio duration (no caption past end-of-audio), (c) the script's word count matches the caption word count, (d) exactly one highlight keyword is flagged per sentence. Read the TLDW `forced_alignment.py` first to learn its alignment backend (local aligner vs. API) and surface any extra dependency/key it needs. Ruff passes.
- **Dependencies:** Sub-phase 1 (needs the rendered audio + the typed scripts from `digests_input.py`)

### Sub-phase 3: Build the Remotion 9:16 digest compositor
- **Files touched:** `remotion/package.json`, `remotion/remotion.config.ts`, `remotion/src/Root.tsx`, `remotion/src/Digest.tsx` (composition), `remotion/src/components/{KenBurnsImage,CaptionTrack,HeadlineCard}.tsx`, `remotion/src/manifest.ts` (the input-props TYPE — the render-manifest contract), `remotion/src/fixtures/sample-manifest.json` (NEW hand-written fixture for standalone dev).
- **What ships:** A Remotion composition `Digest` at 1080×1920 that takes one render manifest and renders: cut 1 = headline card; cuts 2–8 = supplied stills with Ken Burns motion; an overlaid caption track (white, black outline, lower-middle third, word-by-word reveal, one `#FACC15` keyword per sentence); audio from the manifest. Locked structure 0–55s per `reference/stack-notes.md` (0–2s hook … 50–55s loop). Tokens from `reference/design-language.md` (background `#020617`, `caption-highlight #FACC15`).
- **Definition of done:** `npx remotion preview` (Remotion Studio) renders the composition from `fixtures/sample-manifest.json` — manual visual smoke (flagged per Rule 9: UI surface): 1080×1920, ~55s timeline, captions reveal word-by-word with the highlight word in yellow, headline card on cut 1, Ken Burns motion visible. Plus an automated check: a pure mapping function `captionWordsAtFrame(track, frame)` has a unit test asserting the correct words are on-screen at a given timestamp (so caption logic fails the test if wrong, not just if it compiles). `manifest.ts` is the single source of truth for the manifest shape consumed by sub-phase 4.
- **Dependencies:** none (develops against `fixtures/sample-manifest.json`; real audio/captions/images are wired in sub-phase 4). The caption-track field of the manifest must match the JSON shape sub-phase 2 emits.

### Sub-phase 4: Python→Remotion handoff + render the 5 MP4s
- **Files touched:** `agents/pipeline/stages/tts_handoff.py` (PATTERN from TLDW — build the render manifest), `agents/m0/build_render_manifest.py` (NEW — assembles per-digest manifest: audio path + ordered stills from `assets/m0/<digest>/cut-{1..8}.*` + default Ken Burns params per cut + caption JSON + headline-card text), `agents/m0/render_all.py` (NEW — invokes `npx remotion render` per digest), `tests/agents/m0/test_build_render_manifest.py` (NEW). Output: `agents/m0/output/video/digest-{1..5}.mp4`.
- **What ships:** 5 finished `.mp4` digests, each built end-to-end from real audio (SP1) + real captions (SP2) + user-supplied stills, composited by the SP3 Remotion project.
- **Definition of done:** Running `python -m agents.m0.render_all` produces 5 MP4s; for each, `ffprobe` confirms 1080×1920 and duration 45–70s with an audio stream. A unit test on `build_render_manifest` asserts the emitted manifest validates against the `manifest.ts` contract shape (8 image entries, caption track present, headline text present, audio path resolvable) and fails if a digest is missing any of its 8 stills. Manual visual smoke: all 5 play with synced captions + motion + anchor-duo audio. Ruff passes.
- **Dependencies:** Sub-phase 1 (audio), Sub-phase 2 (captions), Sub-phase 3 (composition + manifest contract). Requires the 40 stills present under `assets/m0/` (see Open questions).

## Phase-level definition of done
`agents/m0/output/video/digest-{1..5}.mp4` — five playable files, each `ffprobe`-verified at 1080×1920 / 45–70s / with an audio stream, each showing anchor-duo narration (ALEX=`Leda`, JORDAN=`Sadaltager`), 8 cuts with Ken Burns motion, word-by-word captions with one yellow highlight per sentence, and a headline card on cut 1. `/run-phase` validates the five files + the `ffprobe` checks + the SP1/SP2/SP4 unit tests pass; SP3 is manual visual smoke (flagged). **The audience test itself (do strangers watch 10 in a row?) is the manual M0 go/no-go gate that follows this phase — it is the reason the phase exists but is out of code scope.**

## Out of scope
- News ingestion, ranking, Supabase persistence, Trigger.dev scheduling, the swipe reel UI, Capacitor shell, the bias/trust layer, Story Detail, typed Q&A, and voice mode — all M1+.
- A real image-sourcing service (Pexels/Unsplash/Pixabay/AI-gen/Puppeteer). For M0 the 40 stills are **supplied manually** by the user from the prompts already in `documents/m0-digests.md`; the code only consumes an ordered folder of images.
- Render runtime is **local** (`npx remotion render` on the dev machine) — not Remotion Lambda or a Trigger.dev task (deferred to M1, master-plan Open Q5).
- Parsing `documents/m0-digests.md` programmatically — the 5 scripts are transcribed once into `agents/m0/digests_input.py` (markdown parsing is brittle and out of scope).

## Self-critique

**Product lens:** PASS. M0 is not the MVP — it is the brief's mandated de-risk gate, and it tests the single riskiest assumption (digest quality: do static images + two voices feel too cheap to watch 10 in a row?) *first*, before any M1–M4 build. No scope creep: every sub-phase serves producing the 5 digests; nothing reaches into reel UI, trust layer, or interrogation. The relevant M0 success signal ("would watch ~10 consecutively") is captured as the manual gate in the phase DoD; the 90-day 3x/week metric is correctly an M4 concern and not forced in here.

**Engineering lens:** PASS. All files sit inside the master-plan stack — Python `agents/` (TTS/alignment ports) and a new `remotion/` TS project; no service the plan doesn't include is introduced (no Supabase/Trigger at M0 — render is local, which the plan permits as the spike path). Each DoD is verifiable by a fresh-context agent: SP1/SP2/SP4 via concrete unit tests + `ffprobe` assertions on dimensions/duration/audio-stream; SP3 via a pure `captionWordsAtFrame` unit test plus an explicitly-flagged manual visual smoke. Sub-phase 4 is pure integration/render and cements nothing that must stay flexible — M0 output is a throwaway artifact, and the only contract it fixes (the render manifest in `manifest.ts`) is exactly the seam M1 will want anyway. No two sub-phases are secretly the same: TTS (audio) ≠ alignment (caption timing); composition (TS timeline) ≠ handoff+render (Python manifest + invocation).

**Risk lens:** PASS with one flag. File boundaries are disjoint — `agents/voice/*` (SP1), `agents/pipeline/stages/forced_alignment.py` (SP2), `remotion/src/*` (SP3), `agents/pipeline/stages/tts_handoff.py` + `agents/m0/build_render_manifest.py` + `render_all.py` (SP4); the shared `agents/m0/` dir holds a different file per sub-phase, no collisions. Test coverage: SP1, SP2, SP4 carry real unit tests that fail on wrong business logic (voice mapping, monotonic timings within audio bounds, manifest completeness); **SP3 is manual visual smoke** for the visible composition (acceptable for a UI/video surface per Rule 9, but flagged). Painting-into-a-corner: simulating SP1→2→3→4, SP3 defines the manifest contract against a fixture and SP4 fills it with real data — they meet on the `manifest.ts` shape, and SP2's caption JSON is explicitly required to match that shape, so SP4 still works given the state SP1–3 leave. **Irreversible sub-phases:** none — all output is local files (audio/captions/MP4); no DB migration, no public API, no data deletion.
