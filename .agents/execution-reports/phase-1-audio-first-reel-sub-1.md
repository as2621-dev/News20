## What was implemented

The Next.js 15 static-SPA frontend now exists (there was none before). It static-exports
to `out/`, renders an empty iPhone dev shell (393×852) with the blip wordmark, and exposes
every design token from `prototype/index.html` / `reference/design-language.md` as a working
Tailwind utility class. A Vitest token-config test guards the token contract; Biome lints the
TS/CSS; the export is clean.

- **Dependency set** ported from TLDW `voice-agent-dashboard/package.json` (Next 15, React 19,
  Tailwind 4, radix-ui, framer-motion, zustand, sonner, supabase, zod). Added `vitest` +
  `@vitejs/plugin-react` + `jsdom` (test) and `@biomejs/biome` (lint). Dropped TLDW's
  Trigger.dev/Pinecone/OpenAI/Resend/lucide/next-themes/testing-library/playwright — out of
  scope for SP1. Scripts: `dev`, `build`, `start`, `lint` (biome), `test` (vitest run).
- **Static export** wired (`next.config.ts`: `output: "export"`, `images.unoptimized: true`).
- **Tokens** in `tailwind.config.ts` as an importable `tokens` object (so the DoD test can
  assert keys) AND wired into `theme.extend` so they resolve as utilities. Tailwind 4 reads the
  TS config via `@config "../../tailwind.config.ts"` in `globals.css`.
- **Root layout** self-hosts Inter / Playfair Display / JetBrains Mono via `next/font/google`
  (works under static export), sets `<body>` to `bg-background text-text-primary font-sans`, and
  sets `viewport-fit=cover` via the `viewport` export.
- **`globals.css`** ports the prototype `styles.css` effects needed now — ambient drift
  keyframes, karaoke caption classes (`.caption .w` / `.spoken` / `.active` / `.hl`), scrims,
  the finite-bar + progress classes, the full `.blip` logo CSS, the `.device` dev-shell chrome,
  and the `prefers-reduced-motion` disables. **Class names kept byte-compatible with the
  prototype** so SP2/SP3 port cleanly.
- **`BlipLogo.tsx`** — the wordmark; arc radii `[3.2, 6.2, 9.2]` / 46° opening (`k=0.695`,
  `s=0.719`, `cy=7`) / `viewBox="0 0 11 14"` copied VERBATIM from `app.js` `blipSignal`. Markup
  mirrors `blipLogo(px, cls)` byte-for-byte (`bl<span class="bi">ı<i class="tittle">…`). Props
  `size` (px) + `glow` (boolean → `.glow` pulse, honored by the reduced-motion media query).
- **`PhoneShell.tsx`** — 393×852 dev frame (status bar, Dynamic Island, home indicator). Content
  padded by `env(safe-area-inset-top, 59px)` / `env(safe-area-inset-bottom, 34px)`. Isolated +
  documented as browser-only scaffolding (port-map §1 — dropped in the Capacitor build).
- **`tests/lib/tokens.test.ts`** — 8 Vitest assertions on token *values* (Rule 9 tripwire:
  proven to fail when a value is corrupted, not merely on compile).

## Divergences from the plan + why

1. **Added `src/app/page.tsx`** (not in the explicit file list). Rationale: the listed files
   include `layout.tsx` but no page; a Next static export produces no `index.html` without a
   page, and the DoD's structural smoke (item 4) asserts `out/index.html` references the wordmark
   + PhoneShell. It's a minimal placeholder, explicitly handed to SP3 to overwrite. (Rule 2 — the
   minimum to satisfy the DoD.)
2. **Added `jsdom`** to devDependencies (beyond the brief's named test deps). Rationale: the
   donor Vitest pattern uses `environment: "jsdom"`, and SP2/SP3 will test React components which
   need a DOM. Without it, Vitest errored `Cannot find package 'jsdom'`. It's the donor pattern,
   not new scope.
3. **`@biomejs/biome` pinned to `^2.4.16`** (the resolved version) and the `biome.json` `$schema`
   set to `2.4.16` to silence a schema/CLI version-mismatch info. Trivial reproducibility fix.
4. **Did NOT add `"type": "module"` to `package.json`** despite a benign Next build warning
   (`MODULE_TYPELESS_PACKAGE_JSON` reparsing `tailwind.config.ts` as ESM). Rationale: the TLDW
   donor `package.json` does not set it (Rule 11 — match donor conventions), and it could change
   how `postcss.config.mjs` and other configs resolve. The warning is perf-only; build exits 0.
   Flagging in case a later phase wants to opt in.
